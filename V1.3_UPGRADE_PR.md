# v1.3 Ultra-light Upgrade - このリポジトリをv1.3に移行

## Summary

このADS Frameworkリポジトリ自体を v1.2 Hybrid から **v1.3 Ultra-light** にアップグレードしました。

### Why v1.3?

ユーザーが**5個以上のプロジェクト**を管理しているため、v1.3のSkills統合モデルが最適です。

### Changes

**.claude/CLAUDE.md の更新**:
- **Before**: v1.2 Hybrid (244行、730トークン)
- **After**: v1.3 Ultra-light (90行、270トークン)
- **削減率**: -63% (-154行、-460トークン)

### Architecture

**v1.2のアプローチ** (Before):
```
.claude/CLAUDE.md (244行)
└── 全ての情報を含む
    ├── プロジェクト情報
    ├── ワークフロー定義
    ├── Git運用ルール
    ├── セキュリティ
    └── エラーハンドリング
```

**v1.3のアプローチ** (After):
```
.claude/CLAUDE.md (90行)
├── プロジェクト固有情報のみ
│   ├── プロジェクト概要
│   ├── ディレクトリ構造
│   ├── バージョン管理戦略
│   └── ドキュメント規約
└── 参照: example-skills/solo-dev-workflow/SKILL.md (520行)
    └── 共通ワークフロー定義
```

## Usage in Other Projects

このv1.3アップグレード後、他のプロジェクトでも同様に設定可能:

```bash
# 他のプロジェクトでv1.3を使う
cd /path/to/another-project
mkdir -p .claude
cp /path/to/ADS-Framework-/ads-framework-solo-minimal/CLAUDE-ultralight.md .claude/CLAUDE.md
# プロジェクト固有情報を編集
```

Skills (solo-dev-workflow) は一度だけ設定すれば、全プロジェクトで共有されます。
