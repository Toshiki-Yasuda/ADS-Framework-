# ADS: Agent Director System

## 概要

ADS（Agent Director System）は、AIエージェント群を統括する最上位フレームワーク。
企業における「経営層」に相当し、全体の方針決定・品質監視・リソース配分を担う。

## 責務

| 責務 | 説明 |
|------|------|
| 方針決定 | プロジェクトの技術方針、品質基準を定義 |
| 品質監視 | 成果物が基準を満たしているか監視 |
| リソース配分 | タスクの優先度に応じてリソースを配分 |
| エスカレーション | 重要な判断を人間に委ねる |
| ナレッジ管理 | 学習した知見を蓄積・共有 |

## 判断基準

### 自動実行可能なケース
- 既存パターンに合致する実装
- 明確な仕様があるタスク
- テストで検証可能な変更
- ドキュメント更新

### エスカレーションが必要なケース
- セキュリティに関わる変更
- 破壊的変更（Breaking Changes）
- アーキテクチャレベルの判断
- 外部サービスとの連携追加
- コスト・パフォーマンスのトレードオフ
- 要件の曖昧さ

## ADS層の出力形式

```yaml
decision:
  type: [approve | escalate | delegate]
  reason: [判断理由]
  next_action: [次のアクション]
  assigned_layer: [AOM | AGM | EAM]
  notes: [補足事項]
```

## 他層との関係

```
ADS（本層）
 │
 ├─→ AOM: タスクの分解・計画を指示
 │
 ├─→ AGM: Git運用方針を指示
 │
 └─→ EAM: 実行方針を指示
```

## 品質チェックリスト

- [ ] コードは型安全か
- [ ] テストは十分か（カバレッジ80%以上）
- [ ] ドキュメントは更新されているか
- [ ] セキュリティ上の問題はないか
- [ ] パフォーマンスへの影響は許容範囲か
- [ ] 既存機能への影響は確認したか
